
# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
- リモートリポジトリ
  - ネット上に配置して複数人で共有するためのリポジトリ。(個人開発用にも使える)
- ローカルリポジトリ
  - 開発者一人ひとりが使用するために自分のPC上に配置するためのリポジトリ。

## プッシュとマージの違いは何でしょうか？
- プッシュは、ローカルの編集した内容をリモートにアップする作業で、マージは、別のブランチの作業内容(変更履歴)をブランチに取り込むこと。

## コミットとプッシュの違い
- コミットは変更履歴を残すことで、プッシュはローカルの編集した内容をリモートにアップする作業のこと。


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
- 編集内容をより正確に表すような、一目で変更内容がわかる様なもの。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
- ローカルでマージするフローは、各ブランチでの変更内容を、マスターブランチに変更内容を反映させる。マスターブランチは、各ブランチの変更内容を取り入れる。リモートには反映されない。
- プルリクエストでマージするフローは、変更内容をマージするように依頼する。ローカルには反映されないので、ローカルにプルする必要がある。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
コンフリクトが発生した時、以下の3通りの方法で変更を取り込む
どれを選択するかは状況によって異なるが、特に両方取り込む場合は注意が必要。
1. 先にマージされた変更内容を取り込む
2. 後にマージしようとしている変更内容を取り込む
3. どちらの変更内容も取り込む
